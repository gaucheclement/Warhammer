# Audit des fonctionnalités : Comparaison racine vs warhammer-v2

## Objectif

Analyser toutes les fonctionnalités existantes dans la racine du projet et vérifier leur présence dans la nouvelle version (warhammer-v2). Cette analyse permettra d'identifier les fonctionnalités manquantes et de créer des issues pour les implémenter.

## Périmètre de l'analyse

### Fichiers à analyser dans la racine
- **Code.js** : Logique backend principale (83 KB)
- **Fichiers HTML** :
  - Interface utilisateur (Character.html, CharacterGenerator.html, Admin.html)
  - Gestion des données (Data*.html pour Books, Careers, Skills, Spells, etc.)
  - Générateur par étapes (Step*.html pour Species, Careers, Skills, etc.)
  - Helpers (Helper.html, DataHelper.html, DescriptionHelper.html, EditHelper.html)
  - Menus et navigation (MainMenu.html, Menu.html, NewPage.html, Page.html, Option.html)
  - UI et glossaire (Glossaire.html, Stylesheet.html, JavaScript.html)
- **extract-data.js** : Script d'extraction de données
- **Répertoire data/** : Structure de données existante

### À comparer avec warhammer-v2
- **lib/** : Nouvelles bibliothèques
- **config.js** : Configuration
- **extract-data.js** : Version mise à jour
- **README.md** : Documentation
- **data/** : Nouvelle structure de données
